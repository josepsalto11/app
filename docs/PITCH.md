# PuntualIA Pro — Pitch (versión texto)
## 1) Problema
La gente llega tarde por una combinación de:
- subestimar el tiempo real (tráfico, incidencias, esperas),
- no contar “tiempos invisibles” (preparación, bajar, aparcar),
- y no tener un asistente que convierta agenda → acción ("sal ahora").

## 2) Solución
PuntualIA Pro conecta Calendario + Ubicación y genera automáticamente:
- mejor modo (auto) y rutas multimodales,
- hora exacta de salida con margen inteligente,
- avisos escalonados,
- aprendizaje personalizado (margen óptimo por ruta/día).

## 3) Diferenciadores
- Semáforo de puntualidad (verde/ámbar/rojo) con explicación.
- Márgenes inteligentes aprendidos del usuario (on-device opcional).
- Comparador: más rápida / más fiable / menos estrés.
- Rutinas: Casa→Trabajo, etc. sin configurar cada evento.

## 4) Mercado
- Usuarios urbanos y profesionales (agenda + movilidad diaria).
- Familias/cuidadores: “llega a tiempo a médico”, etc.
- B2B: empresas con puntualidad operativa (visitas, técnicos, logística ligera).

## 5) Modelo de negocio
Freemium:
- Gratis: planificación limitada / recordatorios básicos.
- Pro: ilimitado + IA avanzada + widgets + rutinas + comparador.
B2B: licencias por usuario + analytics (opt-in).

## 6) Roadmap
MVP (4–6 semanas):
- Integración calendario + ubicación
- ETA por modo + cálculo leaveAt
- Notificaciones + navegación externa (Waze/Maps)
- Historial simple + ajuste manual de margen

Pro (8–12 semanas):
- Aprendizaje automático de márgenes
- Background location, geofencing
- OpenTripPlanner + GTFS en ciudades clave
- Widgets y atajos

## 7) Métricas clave
- % eventos puntuales
- minutos ahorrados (menos “por si acaso” excesivo)
- tasa de activación (conectar calendario + 1ª notificación)
- retención semanal (usuarios con rutinas)

## 8) Ask
- 1–2 devs (mobile + backend) + 1 diseñador.
- Acceso a APIs (Directions) o despliegue OTP/GTFS.
